# Paper Monitor 0.1.8

This Windows update adds zero-resident background monitoring while improving refresh accuracy, security, storage efficiency, and startup performance.

## Windows downloads

- `Paper-Monitor-Windows-0.1.8-Setup.exe`: normal per-user installer using the faster onedir application layout.
- `Paper-Monitor-Windows-0.1.8.zip`: portable onedir application bundle.
- `Paper-Monitor-Windows-0.1.8.exe`: standalone onefile executable.
- `SHA256SUMS-0.1.8.txt`: SHA-256 checksums for all Windows artifacts.

## Highlights

- Windows Task Scheduler starts one short-lived refresh worker only when a scan is due; the worker exits after retrieval, storage, and notifications.
- Scheduled definitions are account-isolated, repaired when their operational settings drift, and retry twice at 15-minute intervals after a failure.
- Scheduled notifications are committed to a SQLite outbox before delivery, so a failed toast remains pending for the next retry instead of being lost.
- Scheduled workers defer Dashboard HTML generation until the window is opened, reducing background CPU, memory, and disk work.
- Closing the main window releases the WebView, Python runtime, and local bridge immediately, leaving no Paper Monitor process resident between scheduled scans.
- **Background Monitoring** replaces login/tray startup semantics while retaining the existing configuration key for compatibility; upgrades remove the old `HKCU` Run entry.
- Uninstall removes both the scheduled refresh task and legacy Run entry while preserving user configuration and history.
- `Refresh Now` runs asynchronously and reflects the actual running, succeeded, partial, or failed state, including per-source failures.
- Dashboard refreshes preserve the current Settings or Keyword Analysis view and scroll position.
- Candidate history is written in one SQLite transaction with WAL, indexes, migration tracking, and interrupted-run recovery.
- Local Dashboard requests validate the Host header, authenticate before reading bounded request bodies, and apply browser security headers.
- Crossref cache retention is bounded, and Dashboard writes are atomic.
- Window and tray child processes launch without black console flashes.
- One `Paper Monitor` launch entry replaces the separate Settings shortcut; Settings remains available in the app.
- Installer and ZIP builds use onedir for faster startup, while the standalone EXE remains onefile.

## Signing

Local/manual artifacts may be unsigned when no trusted Authenticode certificate is configured. Public tag releases now require a valid signing certificate and are prepared as drafts before publication.
