# Paper Monitor 0.1.6

Windows reliability and release hardening update.

## Windows Downloads

- `Paper-Monitor-Windows-0.1.6-Setup.exe`: normal per-user installer.
- `Paper-Monitor-Windows-0.1.6.zip`: portable application bundle.
- `Paper-Monitor-Windows-0.1.6.exe`: standalone executable.
- `SHA256SUMS-0.1.6.txt`: SHA256 checksums for all Windows artifacts.

## Highlights

- Dashboard and Settings now share one native Windows window across tray actions and repeated launches.
- Startup routing, runtime tray visibility, refresh serialization, and no-console error logging are more reliable.
- Network sources accept only HTTP(S), enforce response-size limits, and reject XML DTD/entity declarations.
- The repository restores its full cross-platform regression suite and adds Ruff, Bandit, dependency auditing, and complete Windows release automation.

## Verification

- 248 Python tests passed.
- Ruff, Bandit, and dependency vulnerability checks passed.
- Installer, portable ZIP, and standalone EXE hashes were independently verified.
- Standalone and ZIP executables completed the `--help` smoke test with exit code 0.

## Signing

The Windows artifacts in this release are not Authenticode-signed because no trusted code-signing certificate is currently configured. The source and GitHub Actions release workflow support certificate-based signing for future releases.
