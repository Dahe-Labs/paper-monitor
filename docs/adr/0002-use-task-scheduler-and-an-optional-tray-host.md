# Use Task Scheduler and an optional Tray Host

Windows Task Scheduler will own two per-user tasks: `PaperMonitor Refresh` for periodic one-shot refreshes and `PaperMonitor Tray` for starting the optional Tray Host at login. The Tray Host may remain resident only to display the menu and launch finite commands; its **Refresh now** action starts a one-shot background refresh without opening a window, eligible new Articles may notify, and disabling the tray leaves no resident Paper Monitor process. Installed builds use their stable executable path, while portable builds use their current absolute path and reconcile both tasks on every interactive launch; moving a portable folder therefore pauses the tasks until the next launch from the new location rather than copying a hidden runner into AppData.
